# Whack-a-Mole Game 🔨🐹

A simple, interactive Whack-a-Mole game developed during my **6th Semester** as a practical exercise to master Java desktop development. Moles pop up at random positions on the screen, challenging the player to click them as quickly as possible to rack up points before the timer runs out!

This project demonstrates fundamental concepts of GUI creation, event-driven programming, and basic game loops in Java.

---

## 🚀 Features 🕹️

* **Dynamic Gameplay:** Moles appear and disappear at randomized coordinates and intervals.
* **Real-Time Score Tracker:** Points instantly update with every successful whack.
* **Basic Game Logic:** Built-in countdown mechanics using background timers.
* **Event Handling:** Responsive mouse click detection for seamless interaction.

---

## 🛠️ Tech Stack & Tools 🔧

* **Language:** Java
* **GUI Framework:** Java Swing & AWT
* **IDE:** Eclipse / IntelliJ IDEA / NetBeans

---

## 📐 Concepts Learned & Applied 🧠

* **Graphical User Interface (GUI):** Structuring layouts, containers, and interactive button grids.
* **Java Timers (`javax.swing.Timer`):** Managing asynchronous tasks to handle game intervals without freezing the UI thread.
* **Event-Driven Programming:** Implementation of `ActionListener` and `MouseListener` interfaces.
